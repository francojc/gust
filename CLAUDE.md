# gust — Development Project

A terminal-based weather dashboard written in Rust. Renders live
weather data as high-resolution Braille graphs inside a ratatui TUI,
with Vim-style key bindings and offline caching.

## Commands

```bash
# Build
cargo build

# Run in development
cargo run

# Watch for changes
cargo watch -x run

# Run tests
cargo test

# Run tests with snapshot review
cargo insta review

# Release build
cargo build --release

# Install locally
cargo install --path .

# Enter Nix dev shell
direnv allow
# or
nix develop

# Lint
cargo clippy

# Format
cargo fmt
```

## Architecture

The app follows a single-threaded TUI loop with async data fetching
via Tokio channels (`mpsc`). The pattern:

1. `AppState` holds all mutable state.
2. Events and fetch results arrive as `Message` variants.
3. `state.update(msg)` mutates state.
4. `state.view(frame)` renders the current state to the terminal.

Data flows: geocoding → lat/lon → weather fetch → (alerts + AQI
fetches in parallel) → channel → `update()` → `view()`.

## External APIs

- **Open-Meteo** (no key) — forecast data and AQI
- **Open-Meteo geocoding** (no key) — location search
- **NWS alerts** (no key, US only) — weather alerts

## Configuration

`~/.config/gust/config.toml` — units (imperial/metric), theme,
time format, refresh interval, cache duration, default location.

## Testing

- Framework: `cargo test` + `insta` for snapshot tests
- Snapshots live next to source in `src/**/__snapshots__/`
- Mock data in `src/api/mock.rs`
- Use `cargo insta review` after adding/changing snapshots

## Known Gotchas

- After Nix toolchain changes, run `cargo clean` if builds fail
  unexpectedly — stale caches are a common culprit.
- The NWS alerts API returns 404 for non-US coordinates; the client
  silently returns an empty vec rather than erroring.
- Snapshot test filenames are auto-generated by insta; committing
  `src/**/__snapshots__/*.snap` is intentional.
- `cargo-insta` must be installed (via flake.nix) for snapshot
  review workflow to work.
